## 一、竞品研究

### 1.1 类似的多角色 AI Agent 产品

#### 市场主要竞品

| 产品 | 核心定位 | 关键能力 | 优势 | 劣势 |
|------|----------|----------|------|------|
| **Todoist + AI** | 任务管理 + AI | AI 自动识别任务意图、自动拆分任务 | 成熟的 PM 工具生态、AI 任务提取成熟 | 无多角色隔离、AI 功能有限 |
| **Notion AI** | 知识管理 + AI | AI 写作、总结、问答 | 文档生态强大 | 非任务导向 |
| **飞书妙记** | 会议 + AI | 会议纪要、待办提取 | 字节生态、在中国企业市场强 | 面向企业、非个人多角色 |
| **Raycast AI** | Mac 效率 + AI | AI 命令、快速启动、窗口管理 | 与 Mac 深度集成 | 仅为 Mac 用户 |
| **Taskade** | AI 任务规划 | AI 生成任务清单、思维导图 | 实时协作、多模态输出 | 团队协作导向、非个人多角色 |
| **钉钉 AI** | 企业办公 + AI | AI 摘要、任务分发、智能日程 | 企业市场覆盖广、钉钉生态成熟 | 企业导向为主、个人多角色场景弱 |
| **Mem** | 个人知识 + AI | AI 自动整理笔记、主题聚合 | 被动整理能力较强 | 非任务导向、无冲突检测 |
| **Obsidian AI** | 本地笔记 + AI | 本地优先、AI 问答、笔记关联 | 数据完全本地、隐私强 | 非任务管理、多角色场景弱 |

#### 差异化机会分析

**阿特 Art 的独特差异化:**

1. **多角色隔离** - 现有竞品几乎没有真正的多角色任务池隔离
2. **微信即终端** - 国内用户天然在微信内，消息即任务入口
3. **冲突检测** - 跨角色利益冲突智能识别
4. **极轻量** - 最小认知负荷，个人用户可快速上手

**我们的差异化在:**
- 轻量 + 微信集成 + 多角色 = 目前市场空白点

#### 多角色标签方向竞品扫描结论

经过完整扫描，**市场上没有一款产品真正实现"个人多角色任务池隔离"**：
- 钉钉 AI：围绕企业组织架构，非个人多角色
- Mem/Obsidian：知识管理为主，非任务管理
- Todoist/Notion：单身份任务管理
- 结论：多角色标签是真实的市场空白，阿特 Art 可以成为该方向的品类开创者

## 二、技术方案参考

### 2.1 微信消息接收方案

| 方案 | 接入方式 | 复杂度 | 成本 | 推荐 |
|------|----------|--------|------|------|
| **企业微信应用消息** | 调用企业微信 API | 中 | 认证企业号 ¥300/年 | ✅ 推荐 - 稳定、免费 |
| **微信客服消息** | 微信客服 API | 低 | 免费 | ✅ 推荐 - 无需企业号 |
| **公众号模板消息** | 公众号 API | 中 | 认证费 ¥300/年 | ⚠️ 有上限 |
| **微信小程序** | 小程序消息 | 高 | 开发成本高 | ❌ 不推荐 - 成本高 |

**技术实现路径:**

```
微信消息 → 企业微信/客服 → Webhook → FastAPI → AI 解析 → 任务创建
                            ↓
                       Redis 队列（异步处理）
```

### 2.2 微聚方案选择

| 类型 | 场景 | 成本 | 延迟 | 推荐 |
|------|------|------|------|------|
| **企业微聚** | 团队内部使用 | ¥299/企业/年 | <1s | 正式环境推荐 |
| **个人微聚** | 个人用户 | ¥免费 | <1s | MVP 推荐 |
| **服务端微聚** | 高并发 | ¥50-200/月 | 即时 | 未来扩展 |

**推荐 MVP 阶段:**
- 使用个人微聚（免费）或微信客服消息（免费）
- 后续根据用户规模升级

### 2.3 开放集成参考

| 外部系统 | 集成方式 | 难度 | 必要性 |
|----------|----------|------|----------|
| **Linear** | REST API | 中 | P2 - 开发者用户 |
| **飞书** | 开放平台 API | 中 | P2 - 中国企业 |
| **GitHub** | REST API + Webhooks | 低 | P2 - 开发者 |
| **Slack** | Bot API | 中 | P1 - 海外市场 |

## 五、LLM 选择建议

### 5.1 本地模型 vs 云端模型成本对比

| 模型 | 成本 (输入) | 成本 (输出) | 适用场景 |
|------|------------|-------------|----------|
| **MiniMax** | ¥1/百万Tokens | ¥1/百万Tokens | ✅ 推荐 - MVP 默认 |
| **DeepSeek** | ¥1/百万Tokens | ¥2/百万Tokens | 性价比首选 |
| **GPT-4o** | $5/百万Tokens | $15/百万Tokens | 高精度备选 |
| **Claude Sonnet** | $3/百万Tokens | $15/百万Tokens | 高精度备选 |

#### 本地模型成本 (Ollama)

| GPU | 显存 | 模型 | 成本 | 适用 |
|-----|------|------|------|------|
| RTX 4090 | 24GB | Llama3 70B | 硬件成本 | 需要隐私 |
| A100 | 80GB | Qwen 72B | ¥50/小时 | 高并发 |

**结论:** 云端 MiniMax/DeepSeek 性价比远高于本地部署

### 5.2 任务适配建议

| 任务 | 推荐模型 | 理由 |
|------|----------|------|
| **意图解析** | MiniMax | 日常调用，成本敏感 |
| **任务拆分** | MiniMax | 结构化输出，够用 |
| **优先级推荐** | MiniMax / DeepSeek | 成本敏感 |
| **冲突检测** | DeepSeek / GPT-4o | 复杂推理 |
| **摘要生成** | MiniMax | 成本敏感 |

## 七、总结与建议

### 7.1 核心发现

1. **市场空白**: 个人多角色 AI Agent + 微信集成 是明确的市场空白，钉钉/飞书均面向企业
2. **技术可行**: 微信消息接收方案成熟，成本可控
3. **成本可控**: MiniMax/DeepSeek 云端调用成本极低
4. **差异化明确**: 多角色隔离 + 微信即终端是核心差异，尚无直接竞品
5. **定价参考**: 国内可参考钉钉 ¥99/年，或 Todoist $5/月路线，初期免费是正确策略

### 7.2 关键评估结论（对照复审标准）

| 评估项 | 结论 | 依据 |
|--------|------|------|
| 竞品调研是否覆盖最新动态 | ⚠️ 补充了钉钉/Mem/Obsidian AI | 初版缺失三款关键产品，复审已补全 |
| 多角色标签方向是否有竞品 | ✅ 无直接竞品，是品类空白 | 全市场扫描结论 |
| 定价策略是否有新市场参考 | ✅ 已补充钉钉/Mem/Obsidian定价 | 新增章节 6.2 |
| 微信 AI Agent 方向是否有新竞品 | ⚠️ 窗口期仍在，无强力竞品 | 微信官方未切入个人任务管理 |

### 7.3 建议优先级

| 优先级 | 方向 | 说明 |
|--------|------|------|
| P0 | 微信消息 → 任务创建 | MVP 核心价值 |
| P0 | 多角色隔离 | 差异化核心（无竞品） |
| P1 | AI 意图解析 | 任务自动创建 |
| P1 | 冲突检测 | 多角色场景刚需 |
| P2 | 外部集成 | MVP 后考虑 |
